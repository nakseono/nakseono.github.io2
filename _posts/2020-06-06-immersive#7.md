# Immersive #7

immersive 코스의 마지막 스프린트, AWS를 통한 배포가 시작됐다. (내일이 마지막인것 같지만..)

 

AWS에서 주로 사용하게 될 S3, EC2, RDS를 알아보자.

## S3
Simple Storage Service - 인터넷용 스토리지 서비스


언제 어디서나 원하는 양의 데이터를 저장하고 검색할 수 있도록 해준다.

### 몇 가지 용어
- 버킷 : S3에 저장된 객체에 대한 컨테이너이다.

모든 객체는 어떤 버킷에 포함된다.

- 객체 : S3에 저장되는 기본 개체로서, 객체 데이터와 메타 데이터로 구성된다.

데이터 부분은 S3에서 볼 수 없다.

메타 데이터는 객체를 설명하는 이름-값 페어의 집합이다.

- 키 : 버킷 내 객체의 고유한 식별자이며, 버킷 내 모든 객체는 정확히 하나의 키를 갖는다.

S3는 "버킷 + 키 + 버전" 과 객체 자체 사이의 기본 데이터 맵으로 간주할 수 있다.

## EC2

AWS 클라우드의 가상 서버이다. 

컴퓨터를 대여해서 서버로 사용한다고 생각하면 편할 것 같다.

우리는 EC2에 nodeJS 환경을 구축하고 운영 서버를 가동시키면 사용자들이 접속하게 된다.

## RDS

Amazon Realational Database Service - 즉, 데이터베이스 이다.

클라우드에서 관계형 데이터 베이스를 간편하게 설정, 운영 및 확장할 수 있다.

 

굳이 로컬 환경에서 DB를 관리하지 않는 이유는 개발환경(mac os, window, linux 부터 개발 툴 등등..) 이 다들 달라서

 

한 개발자의 PC에서 DB가 구동이 된다고 다른 개발자의 PC에서 구동이 된다고 장담할 수 없다.

 

클라우드 데이터베이스(RDS)를 사용하면 항상 동일한 환경에서 DB를 구성할 수 있기 때문에 사용된다.
